## 🧠 Key Concepts to Master

### Seven Testing Principles
1. **Presence, not absence** - Testing shows defects exist, can't prove they don't
2. **Exhaustive testing impossible** - Must use risk-based prioritization
3. **Early testing saves money** - Detect defects early in SDLC
4. **Defect clustering** - Most defects in few modules (Pareto)
5. **Pesticide paradox** - Repeated tests become less effective
6. **Context dependent** - No universal testing approach
7. **Absence of defects fallacy** - Bug-free ≠ successful product

### Four Test Levels
- **Component** - Unit testing by developers (white-box, white code knowledge)
- **Integration** - Testing interactions between components
- **System** - Complete system vs requirements (black-box, testers)
- **Acceptance** - User validation of business requirements

### Key Distinctions
- **Testing vs QA**: Testing finds defects; QA prevents them through processes
- **Verification vs Validation**: Verification (specs) vs Validation (user needs)
- **Error vs Defect vs Failure**: Human mistake → Flaw → Observable incorrect behavior
- **Static vs Dynamic**: Analysis vs Execution

## 🔧 Troubleshooting

### Progress Not Showing
1. Check if localStorage is enabled in your browser
2. Try refreshing the page (Ctrl+F5 or Cmd+Shift+R)
3. Check browser console for errors (F12)

### Cards Not Loading
1. Ensure JavaScript is enabled
2. Try opening in a different browser
3. Check file paths are correct

### Navigation Issues
1. All links use relative paths - keep files in same directory
2. Don't rename files
3. Open index.html first to get proper navigation
