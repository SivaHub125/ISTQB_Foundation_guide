Interactive: Test Levels in SDLC

Click each level to see detailed information

🔧 Component Testing (Unit Testing)

Focus
Individual software components, modules, or functions in isolation
Scope
Lowest level of testing, smallest testable unit
Responsibility
Developers (white-box testing)
Typical Coverage
Statement, branch, and path coverage
Base
Detailed design or code

🔗 Integration Testing

Focus
Interactions between components and communication between integrated units
Scope
Multiple components combined, test data flow between modules
Responsibility
Developers and/or testers (can be white or black-box)
Typical Types
Component integration testing, System integration testing
Base
Architecture design, integration plan

🖥️ System Testing

Focus
Entire integrated system against specified requirements
Scope
End-to-end functionality, non-functional testing (performance, security)
Responsibility
Independent test team (black-box testing)
Test Types
Functional, non-functional, regression testing
Base
System specification, requirements

✅ Acceptance Testing

Focus
System compliance with user needs and business requirements
Scope
Real-world user scenarios, end-to-end processes
Responsibility
Users, business analysts, customer representatives
Types
User Acceptance Testing (UAT), Operational Acceptance Testing (OAT)
Base
User requirements, business processes, acceptance criteria
